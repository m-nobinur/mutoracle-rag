# Fault Localizer

Phase 5 implements the transparent aggregation and localization rule from the
final plan. It does not use a learned classifier.

## Aggregation

The package exposes three aggregation strategies under `mutoracle.aggregation`:

- `UniformAggregator`: arithmetic mean over available oracle scores.
- `WeightedAggregator`: convex weighted sum; weights must be finite,
  non-negative, and sum to `1.0`.
- `ConfidenceGatedAggregator`: weighted sum only when enough configured oracle
  scores pass the minimum score gate.

Development defaults are configured in `experiments/configs/dev.yaml`:

```yaml
aggregation:
  strategy: weighted
  weights:
    nli: 0.4
    semantic_similarity: 0.3
    llm_judge: 0.3
  delta_threshold: 0.05
```

The calibrated Phase 5 config lives at `experiments/configs/calibrated.yaml`.
It is generated by:

```bash
uv run python experiments/run_weight_search.py --seed 2026
```

The script uses a deterministic synthetic calibration set until FITS exists in
Phase 6. It writes only configuration and calibration metadata; provider secrets
are intentionally set to `null`.

## Decision Rule

For each query, the localizer:

1. Runs the baseline RAG pipeline.
2. Scores the baseline with the configured oracle set.
3. Applies each canonical mutation operator.
4. For prompt-stage query mutations, reruns the pipeline with the mutated query
  and scores that rerun output.
5. Scores each non-rerun mutated run directly.
6. Computes `Delta_i = Omega_0 - Omega_i`.
7. Computes each stage delta as the maximum operator delta for that stage.
8. Predicts the stage with the largest stage delta unless it is at or below the
   configured `delta_threshold`.

The report schema is the public `FaultReport` contract:

```python
FaultReport(
    stage="retrieval" | "prompt" | "generation" | "no_fault_detected",
    confidence=0.0,
    deltas={"CI": 0.0},
    stage_deltas={"retrieval": 0.0, "prompt": 0.0, "generation": 0.0},
    evidence=[],
)
```

Rejected mutations produce a zero delta and an evidence note. Negative deltas
are preserved in `deltas` and `stage_deltas`, but confidence is computed from
non-negative stage evidence so score-improving mutations do not create spurious
confidence.

## CLI

Credential-free smoke diagnosis uses deterministic fixture oracles:

```bash
uv run mutoracle diagnose
```

Model-backed oracle diagnosis is available when the oracle extras and
OpenRouter credentials are configured:

```bash
uv sync --extra oracles --dev
uv run mutoracle diagnose --real-oracles --config experiments/configs/calibrated.yaml
```
