# Fault Localizer

The transparent baseline localizer implements a mutation-delta rule. It does
not use a learned classifier, which keeps the method auditable but also exposes
a clear limitation: full FITS results show retrieval mutations dominate the
stage-delta signal. Current release analyses therefore track both this
transparent baseline and validation-calibrated localizers over the same eleven
operator-delta features. The current comparison includes transparent max-delta,
explicit no-fault gating, stage-specific thresholds, nearest-centroid
calibration, and a small multinomial logistic classifier.

## Aggregation

The package exposes three aggregation strategies under `mutoracle.aggregation`:

- `UniformAggregator`: arithmetic mean over available oracle scores.
- `WeightedAggregator`: convex weighted sum; weights must be finite,
  non-negative, and sum to `1.0`.
- `ConfidenceGatedAggregator`: weighted sum only when enough configured oracle
  scores pass the minimum score gate.

Development defaults are configured in `experiments/configs/dev.yaml`:

```yaml
aggregation:
  strategy: weighted
  weights:
    nli: 0.4
    semantic_similarity: 0.3
    llm_judge: 0.3
  delta_threshold: 0.05
```

The calibrated config lives at `experiments/configs/calibrated.yaml`. It is
generated by:

```bash
uv run python experiments/run_weight_search.py --seed 2026
```

The script writes only configuration and calibration metadata; provider secrets
are intentionally set to `null`.

## Decision Rule

For each query, the localizer:

1. Runs the baseline RAG pipeline.
2. Scores the baseline with the configured oracle set.
3. Applies each canonical mutation operator.
4. For prompt-stage query mutations, reruns the pipeline with the mutated query
  and scores that rerun output.
5. Scores each non-rerun mutated run directly.
6. Computes `Delta_i = Omega_0 - Omega_i`.
7. Computes each stage delta as the maximum operator delta for that stage.
8. Predicts the stage with the largest stage delta unless it is at or below the
   configured `delta_threshold`.

The report schema is the public `FaultReport` contract:

```python
FaultReport(
    stage="retrieval" | "prompt" | "generation" | "no_fault_detected",
    confidence=0.0,
    deltas={"CI": 0.0},
    stage_deltas={"retrieval": 0.0, "prompt": 0.0, "generation": 0.0},
    operator_status={"CI": {"stage": "retrieval", "applied": True}},
    evidence=[],
)
```

Rejected mutations produce a zero delta, an evidence note, and
`operator_status[operator_id]["rejected"] == True`. This is important for E4:
a zero delta can mean either "the mutation ran and produced no score movement"
or "the operator was rejected by its quality gate." The generated E4 summary
therefore reports applied counts and rejection rates rather than treating all
zeros as equally informative. Negative deltas are preserved in `deltas` and
`stage_deltas`, but confidence is computed from non-negative stage evidence so
score-improving mutations do not create spurious confidence.

## CLI

Credential-free smoke diagnosis uses deterministic fixture oracles:

```bash
uv run mutoracle diagnose
```

Model-backed oracle diagnosis is available when the oracle extras and
OpenRouter credentials are configured:

```bash
uv sync --extra oracles --dev
uv run mutoracle diagnose --real-oracles --config experiments/configs/calibrated.yaml
```

## Current Result

On the full E2 FITS artifact, the transparent max-delta rule reaches 30.0%
overall accuracy and remains retrieval-biased. The calibrated centroid
localizer reaches 86.7%, and the validation-trained logistic localizer reaches
90.0%. Treat the transparent rule as an auditable baseline and the logistic
full-delta localizer as the current release-facing localizer.
